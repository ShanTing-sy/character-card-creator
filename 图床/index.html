✱ 使用教程

📖 使用步骤
⚠️ 声明

📦 配置 GitHub Token

  1. 打开 github.com/settings/tokens
  2. 点击 "Generate new token" → "Tokens (classic)"
  3. 在 "Note" 输入框取个名字，比如 我的图床
  4. 在 "Select scopes" 区域，只勾选第一个 repo（子项会自动全选）
  5. 滑到页面底部，点击绿色的 "Generate token" 按钮
  6. 复制生成的 ghp_... 开头的 Token
  7. 回到本页面，点击右上角 ⚙ 打开设置，把 Token 粘贴到输入框
  8. 点击 "保存"，就可以开始上传了！

📤 上传文件

  • 将文件拖拽到左侧虚线区域，或点击左侧 「📤 上传文件」按钮选择文件
  • 支持所有文件格式，单文件最大 25MB
  • 上传过程会在右侧上方的「上传队列」显示进度
  • 上传成功后，网页链接会自动保存到右侧「已上传」列表

🔗 复制链接

  • 在「已上传」列表中点击 📋 按钮一键复制链接
  • 链接可通过全球加速节点访问，所有平台均可正常显示

💾 配置模板

  • 可以保存多套配置（多个仓库、多个 Token）
  • 点击右上角 💾 打开配置模板管理
  • 点击模板名即可一键切换配置

⚠️ 使用声明与风险承担

🚨 重要提示

本工具仅为个人使用设计，使用前请确保理解以下内容：

1. 数据安全

您的 GitHub Token 只存储在浏览器本地的 localStorage 中，仅用于向 api.github.com 发送请求。本工具不会将您的 Token 发送到任何其他地址。

2. 文件存储与访问

上传的文件会存储到您的 GitHub 仓库中。由于文件名为随机生成的字符串，其他人无法猜测出您的文件链接。但请不要上传包含个人隐私信息的文件。

3. GitHub API 限制

已验证的请求每小时最多 5000 次，单文件大小不得超过 25MB。超限后会暂时无法上传，等待一段时间后自动恢复。

4. CDN 缓存

全球加速节点有缓存机制，新上传的文件可能需要几分钟才能全球生效，您可以直接使用 GitHub raw 链接作为备用。

5. 责任声明

本工具仅供个人使用，使用者应自行承担上传内容的合规性责任。请勿上传非法、侵权或敏感内容。

✅ 总结

随机文件名保护是本工具的核心安全机制。只要不把链接传给不相干的人，您的文件就是安全的。
