# SillyTavern 预设标签模式参考

本文档描述 SillyTavern 预设中常见的 XML 风格标签模式。具体标签名因预设而异，需通过 `preset_analyzer.py` 分析实际预设获取。

## 标签分类体系

### 框架类标签

用于包裹和组织聊天上下文的顶层结构标签：

- **主框架标签**：包裹角色回复的完整内容，通常包含摘要、正文、元信息
  - 常见特征：名称与预设主题相关，成对出现
  - 正则作用：作为深度过滤的锚点 `(?<=<框架标签>...)(?=</框架标签>)`

- **标题标签**：承载章节/场景的元数据
  - 常见格式：`<title>字段1：值1 | 字段2：值2 | ...</title>`
  - 正则作用：美化注入的捕获源，通过 `(.*?)` 捕获各字段

- **摘要标签**：每层楼的内容摘要
  - 常见特征：`<summary>` 或变体
  - 正则作用：样式统一，控制折叠显示

### 内容类标签

承载具体内容的各种标签：

- **推理链标签**：包裹 AI 推理过程
  - 常见名称：`thinking`, `reasoning`, `cot`
  - 正则作用：内容隐藏，在渲染前移除

- **附加内容标签**：多种类型的附加信息
  - 常见：各类具名标签包裹特定的场景/规则/旁白内容
  - 正则作用：内容清除或深度过滤

- **分支/直播标签**：交互式内容分支
  - 正则作用：内容清除

### 结束类标签

标记内容结束的声明标签：

- **完成标签**：标记生成结束
  - 常见名称：`finish`, `end`, `done`
- **免责标签**：免责声明内容
  - 常见名称：`disclaimer`

### 指令类标签

用于传递格式和规则指令：

- **视点标签**：控制叙述视角
- **输出规则标签**：控制输出格式约束
- **序章标签**：标记开场内容

### 特殊类标签

- **用户输入捕获**：`<user_input>` 或变体，用于捕获当前轮用户输入
- **角色标识**：`<user>`, `<char>` 等角色占位符

## 常见 HTML 清除目标

这些不是自定义标签，而是模型输出中可能混入的 HTML：

- `<p style...>` — 带内联样式的段落标签
- `</p>` — 段落闭合标签
- `<!--...-->` — HTML 注释
- `<@>`, `<#>` — 占位符残留
- `<br>` — 冗余换行

## 标签分析要点

分析预设时应注意：

1. **嵌套关系**：哪些标签包裹哪些标签
2. **是否成对**：有开标签必有闭标签
3. **出现频率**：高频标签更可能是核心结构
4. **内容长度**：决定清除/隐藏的优先级
5. **上下文位置**：标签在 prompt 流中的位置决定 placement 设置
