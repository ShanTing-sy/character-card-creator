# 世界书创作方法论

## 世界书的本质

世界书是一本为 AI 角色共建故事世界而编写的**幕后百科全书**和**故事指南**。它在幕后工作，确保 AI 能牢牢记住所有世界规则、关键信息和剧情脉络，让整个故事始终保持一致、合理且沉浸。

**类比理解**：
- 角色预设 = 定义"演员（角色）是谁"
- 世界书 = 定义"舞台（世界）是什么样的" + "剧本（故事）该怎么演"

## 世界书解决的三大核心问题

### 问题一：AI 的"健忘症"——保持世界一致性

**症状**：你设定了"这个世界没有金属"，但 AI 突然让角色掏出一把"铁剑"。

**解决方案**：创建强制触发条目。

```
触发关键词：工具、武器、制造
内容：在本世界中，不存在金属矿物。所有工具、武器和机械均由不同属性的"驱动水晶"雕刻而成。水晶的纯度决定了工具的效能。
```

### 问题二：AI 的"跑题王"——聚焦叙事主线

**症状**：正在紧张逃亡，AI 总把话题引向路边小吃和天气。

**解决方案**：创建场景锚定条目。

```
触发关键词：追捕、逃跑、守卫
内容：当前核心情景是 {{char}} 和 {{user}} 正在被城市守卫追捕，形势危急。对话和描写应围绕隐藏踪迹、规划逃跑路线、应对突发危机展开，营造紧张、急促的节奏感。
```

### 问题三：设定的"信息过载"——实现分层展开

**症状**：庞大背景设定全塞进角色设定，开场白变成历史课。

**解决方案**：将复杂背景拆解为多个信息碎片。

```
条目1（关键词：苍炎骑士团）：直属于皇室的精锐，红色铠甲，纪律严明。
条目2（关键词：静默誓言）：二十年前三大骑士团签署的互不侵犯条约。
条目3（关键词：北境叛乱）：导致苍炎与银翼骑士团决裂的关键事件，真相被皇室掩盖。
```

## 条目的两个核心组件

### 触发词设计

触发词是条目的"检索标签"。对话内容匹配触发词时，条目被 AI 激活并运用。

**设计原则**：

1. **覆盖核心词及其同义词/别称**
   - 示例：`虚空低语、禁忌法术、上古咒文、禁咒`

2. **包含常见问法/句式关键词**
   - 示例：`经济、积分、货币、收入、支出、买、卖、价格`

3. **考虑联想链**
   - 当用户提到"剑"时，应触发武器系统条目
   - 当用户提到"升级"时，应触发等级经验条目

4. **空触发词（keys: []）**
   - 用于常驻条目（constant: true），始终在上下文中
   - 适用于文风指南、视角限制等通用铁则

### 条目内容编写

条目内容是 AI 需要绝对记住并运用的"客观事实"。

**编写原则**：

1. **用陈述句写下不容置疑的事实**，像编写教科书或系统档案
2. **精准、精炼、不模棱两可**
3. **包含具体数值和细节**（"40%税率"而非"较高的税率"）
4. **指明适用范围和边界条件**

### 日常场景文风

```
Keys: 市集、花园、酒馆、竞技场、公会、日常、闲逛、休息、聊天
Content: <安全区日常文风>
非副本场景强制生效，全程保持轻松愉快基调。

【核心基调】平淡自然、从容淡定，如拉家常般娓娓道来。文字清淡隽永，多用短句营造舒缓松弛的节奏。NPC 鲜活接地气，自带幽默感和生活智慧，情感表达含蓄节制。

【语言节奏】短句为主，松弛自然。禁用华丽辞藻堆砌。

示例：「中央市集的香料摊前围了三两个人。摊主是个缺了半颗门牙的老头，正用小铜勺舀着一点异域的香料，凑到客人鼻尖前。风一吹，甜丝丝的香气裹着炭火味，飘出去老远。」

【绝对禁忌】①禁止紧绷压抑恐怖文风；②禁止 NPC 写成冰冷工具人；③禁止空泛无细节的场景描写；④禁止直接解释角色行为与动机。
</安全区日常文风>
Extensions: constant: true, position: before_char, depth: 4, sticky: 5
```

```

【中式恐怖】重民俗、重因果、重压抑感，用日常细节铺垫违和感，拒绝直白血腥。
```

## 实战技巧

### 1. 从"打补丁"开始

不必开局就构建完美世界。在聊天过程中，每当 AI 出现一次"遗忘"或"误解"，就立刻为这个误解点创建一个世界书条目。世界书随着故事推进而自然生长。

### 2. 聚焦"不可动摇的事实"

问自己："关于我的世界，有哪些真相是铁律，绝不容许 AI 自由发挥？"把这些铁律写进世界书，设置 `constant: true` 和 `sticky: 5`。

### 3. 世界书是活的"剧本"

世界书可以随时修改：
- 故事前期：`龙是恐怖的灾厄，见者必死`
- 主角成为龙骑士后：`龙是古老而智慧的种族，可与特定心性之人缔结盟约`

### 4. 条目数量控制

- 小型世界（单场景）：10-20 条
- 中型世界（完整城镇）：30-50 条
- 大型世界（多个区域+完整系统）：50-100 条
- 超过 100 条时考虑按类别拆分文件

### 5. 触发词密度检验

每个核心概念至少要有 2-3 个不同角度/称呼的触发词，确保 AI 在用户使用不同表述时都能正确激活条目。

## 常见误区

| 误区 | 正确做法 |
|------|---------|
| 触发词太少（只有一个） | 至少 2-3 个同义词/别称/相关词 |
| 条目内容写成长篇小说 | 精炼到 150-500 字，如档案般简洁 |
| 所有条目设置 constant: true | 仅铁则级法则才需要常驻 |
| 空触发词条目太多 | 空触发词 = 始终消耗上下文，慎用 |
| 忽略文风条目 | 文风指南是最容易被忽略但影响最大的世界书条目 |
