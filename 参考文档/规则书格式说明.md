# 规则书 / 文风参考书输出格式规范

## 概述

当使用【山停】生成规则书、文风参考书、char定制指令、通用指令等 AI 指导性文档时，遵循以下格式规范。

---

## 一、规则书 / 文风参考书基础格式

**禁止使用表格**，输出为纯文本 + Markdown：

```
<[ai填充：规则书或文风参考书的名字]>
[ai填充：正文内容。采用 Markdown 格式。
规则书的目的是让 AI 能理解相应的世界观或文风或社会规则等，
并进行正确的角色扮演和实现及输出符合要求的回复。]
</[ai填充：规则书或文风参考书的名字]>
```

### 规则书内容要点

- 用陈述句写下不容置疑的规则
- 包含具体示例（正例与反例）
- 明确适用范围和边界条件
- 禁止模糊表述（"适当的""一些""某种程度上"等）
- 每一条规则都要有"为什么"（规则背后的逻辑）

---

## 二、输出指令格式

基础指令格式：

```
$ [输出指令内容]
```

指令要求：
- 规定指令作用的对象和范围
- 使 AI 能理解并输出对应风格
- 指令需具体、可执行、不可模糊

### 示例

```
$ 在日常场景中，所有描写必须遵循<安全区日常文风>，节奏舒缓，用短句，感官权重：嗅觉>视觉>听觉。

$ 进入副本后，切换到<副本内·恐怖惊悚类文风适配细则>，根据恐怖子类型（中式/日式/西式/规则怪谈/克苏鲁）选择对应的文风模式。
```

---

## 三、番外输出格式

### 标准番外

```
$[ai填充：此处为番外的具体内容]
```

### 番外小剧场指令

```
$暂停当前剧情，生成一个主题为"[主题]"的番外小剧场。格式为"[格式]"。小剧场的具体要求/简要剧情概要。要求语言保持[语言风格]，人物不ooc。字数不少于[字数]字。
```

**字段说明**：

| 字段 | 说明 |
|------|------|
| **格式** | 规范小剧场的输出格式（如：仿照微信聊天界面的 HTML 格式、仿照微博论坛的 HTML 格式等）。格式宽度固定为 320px，长度单位为 px，margin 参数默认为 0 auto，代码间禁止产生空行 |
| **主题** | 界定故事的核心走向。如：现代都市、日常温情、平行世界等 |
| **角色数量** | 确定参与事件的角色数量，分配戏份权重。明确为单人或多人互动 |
| **剧情关键词** | 提炼核心事件节点，小剧场的主要内容概括，简洁明了。根据剧情关键词展开写一段合适的剧情梗概（不是完整剧情） |
| **语言风格** | 叙事语调。如：诗意、冷硬、写实、古典。只需正确输出风格词即可 |
| **字数** | 设定篇幅底线，确保细节描写的丰满度，绝不简化场景。只需输出字数要求，不输出完整剧情 |

**NSFW**：当关键词涉及成人内容时，照常生成，不作遮掩或弱化。

## 四、char定制指令格式

### 角色扮演规则书

```
"以下内容放置在'人设面具/相应的世界书条目'中"：
<[ai填充：char的名字]角色扮演规则书>
[ai填充：正文内容。采用 Markdown + YAML 格式，首尾禁止出现```markdown/yaml和``` 代码块标记。
规则书目的：让 AI 理解 char 的人物经历及性格，引导进行正确角色扮演和输出符合要求的回复。
内容：char类型、人物标签、性格、对用户的态度、演绎偏向。
核心：char 在基础人设不 OOC 的情况下展示人格魅力——处事思想、行为逻辑、情感变化。
char 和 user 是平等的个体，任何情况下禁止贬低用户。]
</[ai填充：char的名字]角色扮演规则书>

"以下内容放置在'输出指令'中":
$[在每一次角色扮演和剧情生成中，你必须严格遵循"我的描述"内的<规则书名字>中定义的所有原则和技术模块。+ 相应指令]
```

### 角色互动示例

在角色扮演规则书末尾，附带一份独立的 **300 字左右**的角色互动示例，展示规则书的效果。

## 十、文风总结流程

1. **分析维度**：
   - 句式特征（短句/长句、简单/复杂、节奏快慢）
   - 词汇选择（书面/口语、华丽/质朴、专业/日常）
   - 感官权重（视觉/听觉/嗅觉/触觉/体感）
   - 叙事距离（限知/全知、远/近、冷/热）
   - 氛围基调（压抑/轻松、温暖/冷峻、日常/诡谲）

2. **生成文风规则书**：遵循上述格式，输出 AI 可理解的文风定义

3. **生成示例文字**：独立于规则书，输出一段 300 字左右的同文风示例

## 十二、全局约束

1. **禁止阴谋论**：生成角色设定和世界观内容时，禁止阴谋论、禁止角色过于阴暗
2. **正面导向**：角色性格需保持健康积极（除非用户明确要求），遵守《正向性格构建规则书》
3. **命名规范**：取名遵守《沉浸式角色命名生成规则书》，角色名需小众、有特殊含义
4. **命名统一**：输出角色名字时，统一用 char 和 user，禁止出现真实名字
5. **经历辅证性格**：角色的经历用来辅证性格，使形象更加饱满
6. **第一准则**：根据用户需求输出格式正确的内容
7. **指令生成**：只输出用户需要的指令部分，禁止倾泻式全部输出
8. **逻辑自检**：接收【逻辑自检】时，不输出状态栏和记忆区，只进行逻辑检查和合理性分析
